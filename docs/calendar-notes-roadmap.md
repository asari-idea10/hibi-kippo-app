# 暦注マスター本格化ロードマップ v1

## 目的

日々吉方では、暦注を「今日の吉凶表示」だけで終わらせず、吉方旅、ベストデー検索、PDF鑑定、AI解析JSONの根拠として使う。

そのため、150年分、将来的には200年分へ拡張しても扱えるように、日付ごとの計算結果と暦注の意味マスターを分離する。

## 基本方針

暦注は次の3層に分ける。

| 層 | 役割 | 例 |
|---|---|---|
| calendar_base | 日付の基本情報 | 西暦、元号、曜日、祝日、旧暦、六曜 |
| calendar_note_occurrences | その日に該当する暦注結果 | 十二直=執、二十八宿=女、一粒万倍日=true |
| calendar_note_definitions | 暦注の意味・吉凶・表示文 | 一粒万倍日は始めたことが大きく育つ日 |

日次JSONにすべての文章を埋め込まない。日次側はコードと判定だけを持ち、表示やAI解析用説明は定義マスターから参照する。

## 年数拡張方針

| 段階 | 範囲 | 目的 |
|---|---:|---|
| v0 | 2026-05-16〜2026-06-14 | 現在の30日サンプルで画面とAPIを検証 |
| v1 | 1900〜2050 | 既存スプシ・万年暦・天文計算と照合する本体範囲 |
| v2 | 1900〜2100 | 公開サービスとして長期利用に耐える範囲 |
| v3 | 1900〜2150 | 将来的な拡張候補。必要になったら生成・検証 |

200年化を前提に、巨大な単一JSONではなく年別JSONへ分割する。

例:

```text
src/data/calendar-notes/by-year/1900.json
src/data/calendar-notes/by-year/1901.json
...
src/data/calendar-notes/by-year/2100.json
```

## 優先順位

### Phase 1: 現在の30日サンプルを正式構造へ寄せる

- 十二直・二十八宿・二十七宿は正式採用済み
- `legacyRaw` の一粒万倍日、不成就日、三隣亡、八専、十方暮、天一天上、天赦日を定義マスターへ切り出す
- `/api/calendar-notes` で、日次判定と意味マスターを同時に確認できるようにする
- ベストデー仮スコアの参照元を `legacyRaw` から定義マスターへ移す

### Phase 2: 土用・雑節を独立させる

- 土用期間、間日、土用殺方位を `calendar_note_occurrences` に接続
- 節分、彼岸、八十八夜、入梅、半夏生、二百十日、二百二十日などを雑節マスターへ追加
- 雑節は二十四節気・旧暦・固定日が混在するため、計算根拠を必ず保持する

進捗:

- 土用 v0: 選択年連動の期間、間日、土用殺方位、スプシ比較を開発者画面へ接続済み
- 雑節 v0: 節分、春彼岸、秋彼岸、八十八夜、二百十日、二百二十日を二十四節気マスターから算出済み
- 雑節 v0: 入梅、半夏生を前後の検証済み二十四節気時刻から太陽黄経80度/100度として補間表示済み
- 雑節 v0: 入梅、半夏生の太陽黄経80度/100度をSwiss Ephemerisで直接生成するスクリプトと、生成済みマスターを優先参照する受け皿を追加済み
- 雑節検証: 2026年の入梅・半夏生は国立天文台「二十四節気・雑節」の正本サンプルと日付・時刻が一致したため `verified_external_reference` へ昇格済み
- 雑節検証: 高精度計算サイト「雑節」から1950〜2050年の入梅・半夏生202件を取得し、生成済みマスターと日付差分0件で一致。時刻は同サイトにないため `date_verified` として扱う
- 雑節検証: 国立天文台公開範囲2009〜2027年の入梅・半夏生38件を時刻つき正本として追加。37件は時刻まで一致し `verified_external_reference`、2018年入梅のみ1分差のため差分保留
- 次工程: 2018年入梅の1分差を手元万年暦または別天文暦で確認し、丸め・ΔT・計算エンジン差のどれかを切り分ける

### Phase 3: 1900〜2050年の年別JSONを生成

## 個人別方位: 吉方位ラベル設計 v0

本命殺・的殺で「避けるべき方位」を重ねた後、その人にとって使いやすい方位を表示する工程。まずは点数化せず、`風水計算` シートの相性表をアプリ表示用の上品なラベルに翻訳する。

参照元:

- フォーチューンマイレージマスタ / 風水計算シート
  - `B1:I10`: 本命星ごとの吉方位・凶方位一覧
  - `C12:M24`: 本命星と盤上九星の相性表
  - `A29:F37`: 方位、八卦、目的別の方位意味、反対方位
- フォーチューンマイレージマスタ / 方位参考シート
  - 行動目的ごとの年盤・月盤・日盤・時盤の重みづけ候補

吉方位ラベルの採用方針:

スプシ上の記号は、あくまで開発・検証時の補助表現として扱う。日々吉方のアプリ、PDF、AI解析JSONでは、記号そのものを正本にせず、次の表示ラベルを採用する。

| 表示ラベル | 位置づけ | 表示方針 |
| --- | --- | --- |
| `最大吉方候補` | 自分に強く合う方位候補 | 本命星だけで断定せず、月命星・目的・凶方位重複を確認するまでは候補扱いにする。 |
| `吉方候補` | 使いやすい方位候補 | 日常の移動、旅行、整え行動の候補として表示する。 |
| `比和` | 同気・同質の方位 | 強い開運というより、安定、回復、整える方位として表示する。 |
| `中立` | 吉凶を急いで決めない方位 | 目的、日取り、距離、滞在時間、他の凶殺・吉神で判断する。 |
| `相剋注意` | 慎重に扱う方位 | 使えないと断定しすぎず、通常は避ける・専門判断向きとして表示する。 |
| `凶方位優先` | 注意判定の上書き思想 | 吉方候補があっても、五黄殺、暗剣殺、破、本命殺、的殺、土用殺などを優先して注意表示にする。 |

表示ラベルの候補:

- `最大吉方候補`: 本命星だけで判定する段階では「候補」とする。月命星・目的・凶方位との重なり確認後に `最大吉方` へ昇格できる。
- `吉方候補`: 使いやすい方位。ただし共通凶方位、本命殺、的殺、土用殺が重なる場合は採用しない。
- `比和`: 同じ五行の落ち着いた方位。強い開運というより、整える・保つ・回復する方位として表現する。
- `中立`: 吉凶を急いで決めず、目的、日取り、距離、滞在時間で判断する。
- `相剋注意`: 使えないと断定しすぎず、まずは注意ラベルにする。後続で目的別・命式別の例外を検討する。
- `凶方位優先`: 五黄殺、暗剣殺、破、本命殺、的殺、土用殺などが重なる場合の上書きラベル。

優先順位:

1. 共通凶方位、土用殺、本命殺、的殺を最優先で表示する
2. 天道などの例外的な吉神は、凶方位を単純に打ち消す表示にはしない
3. 吉方位ラベルは「使ってよい断定」ではなく、まず `候補` として表示する
4. 本命星のみで `最大吉方` と断定せず、月命星・目的・盤の重なりまで確認してから昇格する
5. 点数化は根拠が固まるまで後回しにする

九星ビューでの表示案:
