# 共通マスター化設計判断書

Last updated: 2026-07-15

Status: `accepted`, `documentation_only`, `implementation_pending`, `source_review_required`

This document records the Step 4A architecture decision for future common-master work. It intentionally does not implement shared masters, refactor source code, rename terms, or change calculation behavior.

## 1. 目的

Step 4A の目的は、Step 3「重複と矛盾の照合」で確認した一致点と保留点をもとに、次工程の実装境界を明文化することです。

特に重要な判断は次の通りです。

- 現在の算命学計算核は、萬年暦研究台帳から整理した関係構造と強く整合している。
- そのため、Step 4B で最初に行うべきことは大規模な共通化ではなく、既存結果を壊さないための回帰テストである。
- 四柱推命由来の中間語や表を、算命学 UI または正式仕様として短絡的に採用しない。
- 九星・方位神で使う支関係と、命式内の算命学位相法を同一レイヤーへ混ぜない。

## 3. 非目的

この文書は次を行わない。

- `src`、`data`、`tests`、`scripts`、package files の変更
- TypeScript 型、JSON マスター、共通定数、回帰テストの追加
- 既存計算ロジック、URL、UI、表示文言の変更
- 四柱推命の通変星名を算命学 UI へ追加する判断
- 十二運名を算命学 UI へ追加する判断
- 干合、自化干合、合化成立、位相法、支合、害、破、刑、自刑、半合、半会の実装
- 方位神の破・冲・刑と、算命学命式内の位相法を統合する判断
- 九星・方位神・算命学の表示名を横断的に統一する判断

## 4. 保護する計算核

Step 3 の照合結果から、次の領域は「修正対象」ではなく「保護対象」とする。

### 4.1 十大主星 100 対応

現在の算命学ロジックは、四柱推命の通変星名を中間レイヤーとして持たず、次の構造で十大主星を算出している。

```text
日干 × 対象干
→ 十大主星
```

Step 3 では、この 10 × 10 = 100 組が、萬年暦研究台帳から整理した陰陽五行関係と全件一致した。

判断:

- この直接計算構造を維持する。
- 通変星レイヤーは、研究説明候補に留める。
- UI・URL・計算結果へ通変星を追加しない。
- Step 4B では、この 100 組を壊さない回帰テストを優先する。

### 4.2 十二大従星 120 対応

現在の算命学ロジックは、十二運名を中間レイヤーとして持たず、次の構造で十二大従星を算出している。

```text
日干 × 対象支
→ エネルギー値 1〜12
→ 十二大従星
```

Step 3 では、この 10 × 12 = 120 組が、萬年暦研究台帳から整理した十二運対応と全件一致した。

判断:

- 現行の `日干 × 対象支 -> エネルギー値 -> 十二大従星` 構造を維持する。
- 十二運名は、研究上の対応語に留める。
- UI・URL・計算結果へ十二運名を追加しない。
- Step 4B では、この 120 組とエネルギー値を壊さない回帰テストを優先する。

### 4.3 初年・中年・老年と身強弱

現在の初年運・中年運・老年運は、日干と年支・月支・日支の組み合わせに基づく。

```text
日干 × 年支 -> 初年運
日干 × 月支 -> 中年運
日干 × 日支 -> 老年運
```

また、十二大従星のエネルギー合計から身弱・身中・身強を判定している。

判断:

- この入力構造と結果を保護する。
- 将来の共通化で、人体星図配置や身強弱判定の意味づけを共通基盤へ押し出さない。
- 身弱・身中・身強の説明文追加は、引き続きソース確認後の別判断とする。

## 5. 共通基盤候補

Step 4B 以降で共通基盤化を検討できる候補は、評価・吉凶・用途を含まない「構造データ」に限定する。

| Candidate | Scope | Step 4A decision |
| --- | --- | --- |
| 十干型・順序 | 甲乙丙丁戊己庚辛壬癸 | 共通基盤候補。 |
| 十二支型・順序 | 子丑寅卯辰巳午未申酉戌亥 | 共通基盤候補。 |
| 九星番号・略称・正式名称 | 1〜9 と名称 | 共通基盤候補。ただし表示文脈は占術別。 |
| 日干 × 対象干の関係構造 | 十大主星算出の土台 | 回帰テストで固定後、最小共通化を検討。 |
| 日干 × 対象支のエネルギー構造 | 十二大従星算出の土台 | 回帰テストで固定後、最小共通化を検討。 |
| 三合4局の純粋構成 | 申子辰、亥卯未、寅午戌、巳酉丑 | 構成だけ共通候補。用途評価は分離。 |

共通基盤へ移す場合も、最初から大規模な master 移設をしない。既存結果を固定するテスト、正本定義の明記、最小差分の順で進める。

## 6. 算命学専用領域

次の領域は、算命学の計算・表示・解釈に属するため、共通基盤へ押し出さない。

- 陰占・陽占の表示構造
- 人体星図の配置
- 十大主星の人体星図上の配置
- 司令蔵干による星算出
- 蔵干方式の選択
- 初年運・中年運・老年運の表示
- 十二大従星のエネルギー合計
- 身弱・身中・身強判定
- 将来の時柱・大運
- 将来の命式内位相法評価
- 将来の運命ダッシュボード view-model
- 将来のアニメーション表示への接続

これらは、たとえ十干・十二支・五行・支関係を共有していても、九星・方位神の用途と同じものとして扱わない。

## 7. 九星・方位神専用領域

次の領域は、九星気学・方位・方位神の文脈に属するため、算命学命式内の位相法と統合しない。

- 年盤・月盤・日盤
- 吉方位判定
- 方位殺
- 歳破
- 歳刑神
- 天道
- 方位神
- 方位上の破・冲・刑
- 三合局の方位利用
- 三合墓の方位利用
- 候補日ランキングにおける吉凶評価

判断:

- 方位神の「破・冲・刑」と、算命学の命式内「支合・冲・害・破・刑」は、文字が同じでも用途レイヤーが異なる。
- 将来共通化する場合は、まず「干支構造としての組み合わせ」と「占術ごとの評価・用途」を分離する。
- 九星側の既存ロジックを壊す可能性があるため、支関係を一括統合しない。

## 10. 推奨レイヤー構成

将来の共通マスター化は、次の分離を守る。

```text
基礎構造レイヤー
  十干 / 十二支 / 九星 / 五行 / 純粋な組み合わせ

占術別計算レイヤー
  算命学: 十大主星、十二大従星、人体星図、身強弱
  九星・方位: 盤、方位神、吉方位、方位殺

評価・解釈レイヤー
  算命学: 位相法、大運、後天運、ダッシュボード
  九星・方位: 候補日ランキング、吉凶、方位評価

表示・アニメーションレイヤー
  UI、説明文、辞書、視覚表現
```

この分離により、同じ干支構造を使っていても、占術ごとの評価を混ぜずに済む。

## 11. Step 4B 着手条件

Step 4B で実装へ進む前に、次を満たすこと。

- この文書と `docs/mannenreki-common-master-research.md` を参照する。
- 既存の算命学出力を修正目的で扱わない。
- 十大主星 100 対応、十二大従星 120 対応、エネルギー値、初中老年運を回帰テストで固定する。
- 月柱・節入り・干合・自化干合・位相法は、テスト対象や TODO として扱っても、実装しない。
- 方位神の支関係と算命学命式内の位相法を統合しない。
- UI 表示語を変えない。

## 12. Step 4B 推奨順序

推奨する Step 4B は次の順序とする。

1. 回帰テスト追加
   - 十大主星 100 組
   - 十二大従星 120 組
   - エネルギー値
   - 初年・中年・老年
   - 身弱・身中・身強境界
2. 正本定義の明記
   - 既存コード上の正本位置を確認し、docs に記録する。
   - 複数定義がある場合は、削除より先に参照関係を記録する。
3. 最小の共通型・定数設計
   - 十干・十二支・九星など、評価を含まない構造から検討する。
   - まだ大規模移設しない。
4. 結果不変の最小共通化
   - テストが守っている範囲だけを小さく動かす。
   - UI・URL・計算結果が変わらないことを確認する。
5. ダッシュボード・アニメーション連携
   - 計算結果を view-model が受け取り、表示が消費する。
   - アニメーションは計算を所有しない。
