# 方位神レイヤー実装ロードマップ

九星方位カレンダーに方位神を組み込むための設計メモ。

## 基本方針

方位神は九星盤の8方位とは粒度が違うため、内部では24山・角度ベースで扱う。

```txt
目的地の角度
↓
24山（壬・子・癸...）へ変換
↓
8方位へ丸めて画面表示
↓
必要に応じて詳細ページで24山まで展開
```

カレンダー本体には常時すべてを出さない。まずは計算層とAPIを整え、方位詳細・象意ページ・検証画面で表示する。

## 実装済み v0

- `src/lib/direction-mountains.ts`
  - 24山マスター
  - 角度範囲
  - 8方位への丸め
  - 十二支の反対方位・支のシフト
- `src/lib/direction-deities.ts`
  - 方位神レイヤー
  - 年盤系: 歳徳神、太歳神、歳破神、大将軍、太陰神、歳刑神、歳殺神、黄幡神、豹尾神、金神 v0
  - 日盤系: 天一神 v0
- `/api/direction-deities?date=YYYY-MM-DD`
  - 日付別に方位神を返す

## 採用済みの考え方

### 1. 評価ではなくレイヤー

方位神は九星の吉方判定を単純に上書きしない。  
「大きな予定なら強く見る」「近場なら注意に留める」など、目的別ポリシーで扱う。

### 2. 8方位に丸めるが、根は24山

画面では `北東` と表示しても、内部では `丑 / 艮 / 寅` を保持する。  
方位神の精度を保つため、8方位だけで判定しない。

### 3. 検証フラグを残す

方位神は流派差・暦資料差が大きい。  
`ruleStatus` と `verificationStatus` を必ず持たせ、未検証のものはカレンダーの強い除外条件にしない。

## UI方針

### 通常カレンダー

九星盤、年/月/日/時の重なり、天道、土用などを主役にする。

### 方位神ON

主要な方位神だけ短く表示する。

例:

```txt
恵方 甲
歳破 午
天一神 艮
```

### 方位詳細ページ

省略せずに表示する。

```txt
8方位: 北東
24山: 艮
角度: 37.5°〜52.5°
方位神: 天一神
意味: ...
行動翻訳: ...
検証状態: needs_manual_almanac_check
```
