## 3. Product Decision: Shichu Suimei and Sanmeigaku Coexistence

プロダクト方針として、四柱推命と算命学は排他的に扱わない。

両者は十干・十二支・陰陽五行を共有する体系として、名称・表現・解釈の違いを明示しながら併用・併記する。ただし、個別項目の完全一致を推測で断定せず、判定構造、成立条件、参照する柱、意味、表示用途を資料ごとに照合する。

対応状態:

| Label | Meaning |
| --- | --- |
| `equivalent` | 判定構造と成立条件を照合し、同等と確認できた。 |
| `partially_equivalent` | 共有部分はあるが、名称、条件、用途、解釈の一部が異なる。 |
| `shared_foundation` | 十干・十二支・陰陽五行などの共通基盤として扱える。 |
| `shichu_only` | 四柱推命側だけで採用する候補。 |
| `sanmeigaku_only` | 算命学側だけで採用する候補。 |
| `source_review_required` | 対応関係または採用可否を追加資料で照合する必要がある。 |

この段階では、追加資料群の四柱推命・算命学対応を一律に `equivalent` としない。

### Step 5B-1 既存コード照合結果

調査日: 2026-07-15

対象commit: `02aa408985ec452b51b0e7f27042f8699dcdfb72`

この照合は、研究台帳に記録した静的36対応を独立期待値とし、`src/data/calendar-master/by-year/*.json` の全期間の日次データから、本命星相当の `yearKyusei`、節月支の `duplicateMonth`、月命星相当の `monthKyusei` の組を集計して比較したread-only調査である。既存実装から期待値を生成して自己比較していない。

原資料側は「ユーザー提供画像をChatGPTが読解・整理した研究記録」であり、Codexは原画像を直接確認、精読、OCRしていない。本節の一致は研究台帳の転記値と既存暦DBの値の一致を示すもので、原資料との完全一致を保証しない。

#### 36対応と108入力

| Comparison | 一致 | 不一致 | 未定義 | その他 |
| --- | ---: | ---: | ---: | --- |
| 3本命星グループ × 12月支 | 36 | 0 | 0 | 同一グループ内差異 0 |
| 本命星9種類 × 12月支 | 108 | 0 | 0 | 複数出力 0、九星1〜9以外 0 |

値の対応は36件、108件とも `existing_exact_match` と分類する。ただし、この分類は値の対応だけを対象とし、節入り境界、本命星・月命星としての命理上の正式採用、表示・判定用途まで確定するものではない。

#### 現在の実行経路

```text
birthDate
→ 暦DBの出生日行
→ yearKyusei
→ monthKyusei
→ buildPersonalProfile
→ getKeishaProfile
→ 傾斜プロフィール
```

- 独立した月命星算出関数と静的36対応masterは存在せず、状態は `current_code_missing`。
- 月命星相当値は、暦DBの日次既計算値 `monthKyusei` を読むことで暗黙に展開される。
- `monthKyusei` は月盤中宮と月命星相当値で共用されている可能性がある。意味上の正式対応は `source_review_required`。
- `yearKyusei` は現在の個人コンテキストで本命星として仮採用されるが、正式採用は `source_review_required`。
- 値の対応自体は `existing_exact_match` であり、専用関数や静的masterがないことを値の欠陥とは扱わない。

#### 現在の利用先

利用を確認した範囲:

- `/purpose-calendar` の本人プロフィール
- 傾斜、傾斜星、傾斜の良い面・注意面
- 特定サンプルの傾斜方位神

現在の利用を確認できなかった範囲:

- 月命星そのものの直接表示
- 吉方位判定、候補ランク、`strict` / `standard` / `loose`
- 月命殺、月命的殺、および本命殺・的殺以外の月命系判定
- 同行者判定、相性判定
- URL query parameter、calendar notes、用語辞典

未利用項目は欠陥と断定せず、プロダクト仕様未確定または新規候補として扱う。同行者は現在、本命星を直接指定する経路を使い、月命星相当値を計算へ渡さない。

#### 節入り境界

- 12節すべてで月支と月九星が日付単位で切り替わる。
- 月支切替は立春、啓蟄、清明、立夏、芒種、小暑、立秋、白露、寒露、立冬、大雪、小寒を使い、中気を使わない。
- 立春日には年九星、月支、月九星が同時に切り替わる。
- 現在の出生入力は `birthDate` のみで、出生時刻と節入り時刻を出生判定に使わない。
- 現在の精度は日付単位であり、同一日の節入り前後を区別できない。
- 以上は基本構造が一致する一方で時刻境界を扱わないため、`existing_partial_match` とする。

公式節入りmasterはJST分単位の時刻を保持するが、個人星の現在値は暦DBの日次行から取得する。`getSetsuiriContextByDate` も日付で比較しており、時刻境界による出生判定を行わない。
